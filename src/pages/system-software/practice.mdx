#### 1. Напишите cron-задачу для запуска приложения waker каждое утро, в 7:00.

1. Откройте терминал или командную строку.
2. Введите следующую команду:

```shell
crontab -e
```

3. Откроется редактор для редактирования вашего crontab.
4. Добавьте следующую строку:

```shell
0 7 * * * /path/to/waker
```

Здесь:

- `0 7` означает запуск в 7:00 утра.
- `* * *` указывает, что задача должна выполняться ежедневно.
- `/path/to/waker` - это путь к исполняемому файлу вашего приложения `waker`. Убедитесь, что вы указали правильный путь.

5. Сохраните файл и закройте редактор.

#### 2. Разверните контейнер веб-сервера nginx из стандартного репозитория.

1. Установите Docker, если он ещё не установлен на вашем компьютере. Вы можете найти инструкции по установке на официальном сайте Docker.
2. Запустите Docker и войдите в свой аккаунт Docker Hub, если у вас его ещё нет.
3. Создайте каталог для вашего проекта и перейдите в него.
4. Создайте файл с именем `Dockerfile` в корне вашего проекта. В этом файле будут указаны инструкции для создания образа контейнера. Вот пример простого файла `Dockerfile`:

```dockerfile
FROM nginx:latest
COPY ./default.conf /etc/nginx/conf.d/default.conf
```

Этот файл говорит Docker, что нужно взять образ nginx из официального репозитория и скопировать файл конфигурации `default.conf` в контейнер.

5. Создайте файл конфигурации nginx с именем `default.conf` в корне вашего проекта. Этот файл будет определять, как nginx будет обрабатывать запросы. Вот пример простого файла `default.conf`:

```
server {
 listen 80;
 root /usr/share/nginx/html;
 index index.html index.htm;
}
```

Этот файл говорит nginx слушать порт 80 и обслуживать файлы из каталога `/usr/share/nginx/html`.

6. Создайте образ контейнера с помощью команды `docker build`:

```shell
docker build -t my-nginx .
```

Эта команда создаст образ контейнера с именем `my-nginx` на основе файла `Dockerfile` в текущем каталоге.

7. Запустите контейнер с помощью команды `docker run`:

```shell
docker run -d -p 80:80 my-nginx
```

Эта команда запустит контейнер с образом `my-nginx` и откроет порт 80 для внешнего доступа.

8. Проверьте, что контейнер работает, с помощью команды `docker ps`:

```shell
docker ps
```

Эта команда покажет список запущенных контейнеров.

9. Посетите ваш веб-сервер в браузере по адресу [http://localhost](http://localhost/). Вы должны увидеть страницу приветствия nginx.

#### 3. Напишите команду, которая позволяет развернуть docker контейнер PostgreSQL из стандартного репозитория.

Чтобы развернуть контейнер PostgreSQL из стандартного репозитория Docker, используйте следующую команду:

```shell
docker run -d --name postgresql -e POSTGRES_PASSWORD=your_password -p 5432:5432 postgres
```

Эта команда создает контейнер с именем `postgresql`, устанавливает пароль для базы данных PostgreSQL, используя переменную окружения `POSTGRES_PASSWORD`, и открывает порт 5432 для доступа к базе данных.

#### 4. Напишите задачу изменения конфигурации Ansible для установки дополнительного пакета в системе.

Для установки дополнительного пакета в системе с помощью Ansible, создайте файл задач (`tasks/main.yml`) со следующим содержанием:

```yaml
- name: Install additional package
 apt:
 name: <package_name>
 state: present
```

Замените `<package_name>` на имя пакета, который вы хотите установить. Например, чтобы установить пакет `vim`, задача будет выглядеть так:

```yaml
- name: Install vim
 apt:
 name: vim
 state: present
```

Сохраните файл и запустите Ansible, указав имя вашей роли и хоста(ов), на которых вы хотите выполнить установку пакета. Например:

```shell
ansible-playbook -i inventory install_package.yml
```

Убедитесь, что ваш инвентарь (`inventory`) содержит хост(ы), на которые вы хотите установить пакет.

#### 6. Напишите скрипт, который выводит всю информацию обо всех активных системных процессах для всех пользователей.

```bash
#!/bin/bash
ps -aux | grep -v '^\s*root' # Исключаем процессы пользователя root
```

Этот скрипт использует команду `ps` с опцией `-aux` для вывода подробной информации обо всех активных процессах и `grep` с параметром `-v` для исключения строк, начинающихся с пробела и символа `root`, что позволяет избежать отображения процессов пользователя `root`.

#### 7. Опишите команду и ее формат для изменения уровня привилегий для каталога /etc/.

Чтобы изменить уровень привилегий для каталога `/etc/`, можно использовать команду `chmod` с опцией `-R` для рекурсивного изменения прав доступа и опциями `u+rwx,go-rwx` для добавления прав чтения, записи и выполнения для владельца и удаления прав чтения и записи для группы и остальных пользователей.

Пример команды:

```shell
chmod -R u+rwx,go-rwx /etc/
```

Эта команда добавит права чтения, записи и выполнения для владельца каталога `/etc/` и всех его подкаталогов и файлов, а также удалит права чтения и записи для группы и остальных пользователей.

#### 9. Опишите команду и ее формат для добавления репозитория myrepository в пакетный менеджер apt.

Чтобы добавить репозиторий `myrepository` в пакетный менеджер `apt`, вам нужно выполнить следующие шаги:

1. **Найдите адрес репозитория.** Это может быть URL или путь к файлу `.deb`.
2. **Добавьте репозиторий в список источников.** Используйте команду `add-apt-repository` с параметром `--source` и адресом репозитория. Например:

```shell
sudo add-apt-repository --source "адрес_репозитория"
```

3. **Обновите список пакетов.** После добавления репозитория обновите список доступных пакетов с помощью команды `apt-get update`.

#### 10. Опишите команду и ее формат для запуска контейнера в системе docker.

Для запуска контейнера в системе Docker используется команда `docker run`. Вот её основной формат:

```shell
docker run [OPTIONS] IMAGE[:TAG|@DIGEST] [COMMAND] [ARG...]
```

Где:

- **`OPTIONS`** — опции запуска контейнера, такие как указание портов, монтирование томов, установка переменных окружения и т.д.;
- **`IMAGE[:TAG|@DIGEST]`** — имя образа Docker, который будет использоваться для создания контейнера. Можно указать тег (`TAG`) или дайджест (`@DIGEST`);
- **`COMMAND`** — команда, которая будет выполнена внутри контейнера после его запуска;
- **`ARG...`** — аргументы, передаваемые команде.

Вот пример запуска контейнера с веб-сервером Nginx, который будет доступен на порту 8080:

```shell
docker run -p 8080:80 nginx
```

В этом примере опция `-p` используется для проброса порта с хоста на порт контейнера.

#### 11. Опишите возможные способы добавления нового репозитория в репозитории apt с описанием команд и их форматов.

Чтобы добавить новый репозиторий в репозитории `apt`, можно использовать команду `add-apt-repository`. Вот несколько примеров:

1. **Добавление репозитория с помощью URL:**

```shell
sudo add-apt-repository "адрес_репозитория"
```

2. **Добавление репозитория с помощью файла конфигурации:**

```shell
sudo add-apt-repository -f "путь_к_файлу_конфигурации"
```

3. **Добавление репозитория с указанием версии пакета:**

```shell
sudo add-apt-repository "адрес_репозитория" "версия_пакета"
```

4. **Добавление репозитория с указанием ветки:**

```shell
sudo add-apt-repository "адрес_репозитория" "ветка"
```

5. **Добавление репозитория с указанием архитектуры:**

```shell
sudo add-apt-repository "адрес_репозитория" "архитектура"
```

6. **Добавление репозитория с указанием ключа GPG:**

```shell
sudo add-apt-repository "адрес_репозитория" "ключ_GPG"
```

После добавления репозитория необходимо обновить список пакетов с помощью команды `apt-get update`.
